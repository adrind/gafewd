CSS Positioning

CSS Positioning: Overview

• The position CSS property defines alternative positioning rules for elements

• Useful for advanced layouts, fixed headers, and animated elements

• When set, can use the CSS properties: top, right, bottom, left

• position has four values:


	position: static;
	position: fixed;
	position: relative;
	position: absolute;
position: static;

• Default CSS positioning value for ALL elements

• Considered "normal" behavior

• Static elements are NOT positioned

• Cannot set the following properties:

⇒ top

⇒ right

⇒ bottom

⇒ left

position: fixed;

• Positioned relative to the browser window

• Element will stay in view, even as window scrolls

• Removed from the normal flow of the document

• Used for for persistent navigation: "sticky headers"

• Can position using:

⇒ top

⇒ right

⇒ bottom

⇒ left

Sticky Header Example

• Apply the following CSS rules to the element you want to fix:


 header {
   position: fixed;
   width: 100%;
   top: 0;
   height: 50px; // should match the padding-top on body
 }

• Apply the following CSS rules to your <body>:


 body {
   padding-top: 50px; // should match the height of header
 }

position: relative;

• Positioned relative to its normal position

• Move using top, right, bottom, and left properties

• Note: the original space for the element is preserved

• Used for:

⇒ Overlapping elements

⇒ As a reference for absolutely-positioned elements

position: absolute;

• Positioned relative to either:

⇒ Browser window

⇒ Closest positioned element

• Removed from the normal flow of the document

• Element will collapse to content's width

• Used for:

⇒ Animations (we will see shortly)

⇒ Precisely placing an element within a parent

absolute & relative

• Use position: absolute with position: relative to absolutely position an element within another element

• position: relative is placed on the parent element

• position: absolute is placed on the child element

CSS Animations

• Two parts to creating a CSS Animation

⇒ @keyframes - define the animation

⇒ animation CSS property to assign that animation to a selector

@keyframes

• @keyframes is a keyword in CSS that lets us define an animation

@keyframes redToBlue {  
  from {
    background: red;
  }
  
  to {
    background: blue;
  }
}

⇒ redToBlue - the name of our animation

⇒ from - defines the starting style at the beginning of the animation

⇒ to - defines the style at the end of the animation

animation

• Once we define an animation with @keyframes we need to apply it to a selector

.color-block {
  width: 100px;
  height: 200px;
  animation: 3s redToBlue infinite;
}

⇒ 3s - the amount of time it takes to complete the animation (makes it faster or slower)

⇒ redToBlue - the animation we are using

⇒ infinite - tells it to repeat an infinite number of times

JavaScript animation libraries

JavaScript Animations

• There are some JavaScript/jQuery libraries that specialize in animations

• It can be very tedious to write complex animations in CSS.

• <svg> and <canvas> give us a way to make these detailed animations.

• We use JavaScript to interact and change these elements.

SVGs

• Scalable Vector Graphics

• SVG is a language for describing two-dimensional graphics.

• SVG drawings can be interactive and dynamic. Animations can be defined and triggered either declaratively (i.e., by embedding SVG animation elements in SVG content) or via scripting.

• Written in XML (which kind of resembles HTML) it has its own set of special attributes and tags you can use to create shapes.

CSS Layout

CSS Layout: History

• As you are probably aware...HTML/CSS page layout can be difficult

• In the early days, HTML and CSS specs were slow to develop and browsers were being built without any guidance

CSS Layout: Tables

• An early strategy for building complex layouts was using table

• This quickly became known as bad style because it was not semantic -- a table should be used as a table of data, not for layout

• Also poor for accessibility

• Example

CSS Layout: CSS Tables

• The use of tables later inspired CSS Tables

• AKA display: table; and display: table-cell;

• It let us style elements as "tables" so we didn't have to use the table element.

• Not great, inflexible when it comes to responsiveness, semantically still not right.

• Example

CSS Layout: Floats and inline-block

• Floats and inline-block become new standards but are still difficult

• Floats require "clearfix" to make sure it doesn't break out of its container

• display:inline-block; not supported below IE8

• Floats example

• Inline-block exmaple

CSS Layout: Flexbox

• By the early 2010s we still hadn't perfected page layout.

• 2011: A spec for Flexbox was proposed

• Now we have strong support

• Flexbox Example

CSS Layout: Grid

• Flexbox is powerful but not perfect

• Good for arranging items on one axis, but not for 2D layouts.

• A spec for CSS Grid was proposed by Microsoft in 2011

• Currently at decent support despite not being an "official" standard

• CSS Grid Example

CSS Frameworks

CSS Frameworks: Overview

• Standardized approach for solving common problems

• Pre-packaged code used to support web development

• Goals:

⇒ Provide a common structure (boilerplate)

⇒ Eliminate repetitive work

⇒ Speed development process

⇒ Increase on-boarding time of new developers

CSS Frameworks: Overview

Frontend Framework:

• Contains standardized code and assets (HTML/CSS/JS)

• Typography and pre-built styles for HTML elements

• CSS grid for developing a responsive layout

• Solutions to cross-browser compatibility issues

• JS library for UI/UX interactions (dropdowns/carousels/etc.)

CSS Frameworks: Overview

Two types of framework:

1) Grid/Layouts (simple):

⇒ 960 Grid System

⇒ Toast

⇒ Base

2) Complete (complex):

⇒ Bootstrap

⇒ (Zurb) Foundation

⇒ Skeleton

CSS Frameworks: Selecting a Framework

Rule of thumb:

⇒ The more stylistically complex (the more custom CSS you will be writing), the less complex of a framework you will need

As with plugins:

⇒ Open source

⇒ Recently updated

⇒ Community of active contributors

⇒ Well-documented

CSS Frameworks: Pros/Cons

Pros:

⇒ Major increase in speed of development

⇒ Solutions to common problems already built

⇒ Cross-browser compatibility

Cons:

⇒ Reliance: you might un-learn CSS!

⇒ Dependence: what if you want to switch frameworks?

⇒ Bloat: unused framework code

⇒ Identical/similar-looking/clones

Bootstrap

Bootstrap: What is it?

In their own words:

"Bootstrap is the most popular HTML, CSS, and JS framework for developing responsive, mobile first projects on the web."
Bootstrap: What is it?

• Released by Twitter as an open source project in 2011

• OG built by @mdo and @fat

• CSS framework (with HTML and JS components)

• Collection of CSS styles for common UI components:

⇒ Navbars

⇒ Buttons

⇒ Forms

⇒ Image Carousels

• Notable for its responsive grid layout

• Second most starred repo on GitHub!

Bootstrap: What is it?

• Premade CSS and JS files you can add!

Bootstrap: The Grid

Source: Girl Develop It

Bootstrap: The Grid

Core CSS Classes:

.container

⇒ Top-level element; contains other HTML

.row

⇒ Horiztonal row used to hold .col's

⇒ Must be children of a .container

.col

⇒ Used to hold content (text/images/etc.)

⇒ Must be immediate children of row's

Bootstrap: Columns

• Bootstrap's grid is divided into 12 columns

• Columns are created using CSS classes

• Build columns using the following formula: col-{size}-{width}

• <div class="col-md-6">

⇒ A <div> for medium-sized screens that is 6-columns wide

• <div class="col-xs-2">

⇒ A <div> for x-small screens that is 2-columns wide

Bootstrap Components: Navbar

• Bootstrap's Navbar provides a responsive nav out-of-the-box

• Provides built-in links, search, and dropdown components

• Can be "fixed" to top/bottom of screen

• Easy-install:

1) Copy the starter code from the documents

2) Paste the starter code right before the opening <body> tag

3) Customize to your liking
